================================================================================
 CHUBCHAMPIONS — SKILL TREE ICON GENERATION PROMPTS
================================================================================

Three prompts for generating sprite-sheet images of skill icons matching the
ChubChampions UI style: readable hand-painted game icons with a rough fantasy
UI feel. Avoid the glossy "AI mobile ad" look.

WORKFLOW:
  1. Pick an image AI (Midjourney v6/v7 recommended; Flux 1.1 Pro also great;
     DALL-E 3 works for smaller grids).
  2. For each tree, paste the STYLE BLOCK (below) followed by that tree's
     prompt. They are designed to be combined — the STYLE BLOCK is identical
     across all three so the three sprite sheets feel like one set.
  3. Generate at the recommended resolution.
  4. Crop into individual cells, save as
     attack_<id>.png / defense_<id>.png / alchemist_<id>.png.

================================================================================
 STYLE BLOCK — paste this identically at the top of EACH of the three prompts
================================================================================

STYLE — apply identically to every cell:
- Hand-painted 2D fantasy game icon style with visible brush texture, slightly
  imperfect edges, and controlled shape language. It should feel made by a game
  artist, not like a glossy AI-generated mobile ad.
- Use simple readable silhouettes first. Each icon must be recognizable at small
  size without relying on tiny details, text, symbols, or over-rendered effects.
- Painterly shading with 3 to 5 clear value steps, mild ambient occlusion, and
  a restrained rim light. Avoid plastic shine, chrome highlights, excessive
  bloom, lens flare, glitter, and over-saturated neon.
- Use a dark warm-brown ink outline, not pure black. Outline thickness should be
  consistent and slightly uneven, like hand-inked game UI art.
- Subject is centered, fills about 72-78% of the cell, slightly angled only when
  useful for readability. No extreme perspective, no cinematic lighting.
- Per-cell background: muted parchment/stone tile, warm cream to tan
  (#ead6aa -> #b99056), with subtle paper grain and a very soft tint from the
  tree accent color. No radial glow blobs, no glossy button backgrounds.
- Each cell has a simple 1px carved frame in warm dark brown (#6f4c28) separating
  it from neighbors. Clean grid, no bleeding between cells.
- Palette should be cohesive and earthy: warm browns, muted reds/blues/greens,
  small bright accents only where needed for gameplay readability.
- Add tiny hand-painted chips, scratches, dust, or cloth texture when appropriate,
  but keep the composition clean and iconic.

LAYOUT:
- Single image, perfect uniform grid, all cells identical square aspect.
- Read order: top-to-bottom, left-to-right (English reading order).
- No labels, no text, no numbers, no watermarks, no signatures — visuals only.

NEGATIVE / AVOID:
- No glossy AI look, no plastic 3D, no mobile ad render, no smooth airbrushed
  gradients, no excessive bloom, no generic fantasy stock-icon style.
- No dark fantasy, no Diablo, no horror, no realistic photography, no anime.
- No human faces in close-up realism (chibi/cartoon faces OK, profile
  silhouettes OK).
- No gore, no scary imagery — keep it cute even for "death" themed icons
  (small stylized skull, soft red eye glow, etc.).
- No drift in style between cells; every icon must clearly belong to the same
  set.

================================================================================
 TROUBLESHOOTING TIPS
================================================================================

1. If the AI inserts text, labels, or numbers into the icons:
   - Midjourney: append "--no text --no labels --no captions" to the prompt.
   - DALL-E 3 / ChatGPT image: add at the bottom "absolutely no text, letters,
     or numbers anywhere in the image".
   - Flux: include "clean iconography, no typography, no written language" in
     the negative prompt.

2. If the cells visually bleed into each other or subjects span multiple cells:
   - Add: "each cell is visually isolated by a clear 1px hairline border,
     strict grid separation, every subject is contained inside its own cell,
     no overlap between cells".

3. If the bold black outline does not appear:
   - Reinforce: "every subject has a strong 1-2px black ink outline matching
     Cookie Run Kingdom and Disney Magical World icon style".

4. If colors are too muted or dark:
   - Add: "bright cheerful palette, saturated local colors, soft warm
     cream-cream-peach cell backgrounds, family-friendly mobile game look".

5. Coherence test before generating the full sheet:
   - Ask the AI to first generate a 2x2 sub-grid with 4 representative icons
     (e.g. Power Strike, Vitality, Brew Master, plus one more) using the same
     STYLE block. Verify the outline thickness, shading direction, color
     temperature, and background texture are right.
   - Once satisfied, scale up to the full grid for that tree.

6. Final asset workflow after generation:
   - Open the sprite sheet in any image editor (Photoshop, GIMP, Photopea web).
   - Slice into individual cells (the visible 1px hairline frame helps mark
     the cuts).
   - Save each as PNG named:
       attack_power_strike.png  ... attack_predator.png        (15 files)
       defense_iron_hide.png    ... defense_sanguine_ward.png  (22 files)
       alchemist_brew_master.png ... alchemist_synthesis.png   (15 files)
   - Drop them into Assets/UI/skill-icons/ in the ChubChampions repo.
   - Ask Claude to swap SKILL_ICONS in index.html from emojis to <img> tags
     pointing at the new files. ~20-line edit.
